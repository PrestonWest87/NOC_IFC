# Enterprise Architecture & Algorithmic Specification: `src/train_model.py`

## 1. Executive Overview

The `src/train_model.py` module is the **Machine Learning Pipeline and Feedback Loop Engine** for the Intelligence Fusion Center (IFC). It is responsible for training the probabilistic text classification model that powers the Artificial Intelligence "Safety Net" utilized in the `logic.py` scoring engine.

Instead of relying on a static, pre-trained model, this module implements a **Human-in-the-Loop (HITL)** reinforcement architecture. It continuously learns from the explicit actions of NOC analysts. When operators designate an intelligence feed item as "Noise" or "Keep", this script ingests those decisions to train a bespoke, mathematically weighted Logistic Regression model tuned perfectly to the organization's unique threat profile.

---

## 2. Data Ingestion & Preprocessing

### 2.1 The Human-in-the-Loop (HITL) Dataset
The `train()` function begins by opening a database session and querying the `Article` table for ground-truth training data.
* **Filter Criteria:** It strictly pulls articles where `human_feedback` is either `1` (Dismiss/Noise) or `2` (Confirm/Keep). Unrated articles (`0`) are ignored to prevent introducing untagged noise into the training set.
* **Pandas Integration:** The SQLAlchemy query is passed directly into a Pandas DataFrame (`pd.read_sql()`) for highly optimized, vectorized data manipulation.

### 2.2 Volume Safety Threshold
* **Minimum Viable Data (MVD):** Before proceeding, the engine checks if `len(df) < 10`. Machine Learning models require a baseline variance to identify statistical patterns. If the system attempts to train the model with fewer than 10 labeled articles, the script safely aborts, preventing the generation of an overfitted, highly inaccurate model.

### 2.3 Feature Engineering & Label Mapping
* **Feature Concatenation:** The predictive feature (`X`) is generated by concatenating the article's `title` and `summary`, forcing all strings to lowercase. This maximizes the text corpus available to the vectorizer, providing deeper semantic context than a headline alone.
* **Binary Labeling:** The raw database integers are mapped to standard binary classification labels (`y`):
  * `1` (Dismiss) -> `0` (Noise)
  * `2` (Confirm) -> `1` (Important)

---

## 3. The Machine Learning Architecture

The module utilizes `scikit-learn` to construct a highly efficient, two-stage NLP pipeline (`make_pipeline`) that has been significantly upgraded from legacy Naive Bayes implementations.

### Stage 1: Advanced TF-IDF Vectorization
The text data is passed into `TfidfVectorizer`.
* **Mechanism:** Converts the raw text into numerical arrays (vectors). It calculates the Term Frequency (how often a word appears in an article) and multiplies it by the Inverse Document Frequency (penalizing words that appear across all articles).
* **N-Gram Context:** `ngram_range=(1, 2)` allows the model to learn two-word phrases (bigrams) like "data breach" or "buffer overflow", providing vast contextual improvements over single-word analysis.
* **Corpus Optimization:** The `stop_words='english'` parameter strips out low-value conjunctions. Furthermore, `max_df=0.9` algorithmically ignores terms appearing in over 90% of documents (corpus-specific stop words), ensuring only high-value threat terminology is vectorized.

### Stage 2: Logistic Regression Classifier
The vectorized data is fed into a `LogisticRegression` classifier.
* **Selection Rationale:** Logistic Regression is utilized because it provides highly accurate, well-calibrated probability scaling (`predict_proba`), which is strictly required by the `logic.py` Dynamic Override Matrix.
* **Class Balancing:** `class_weight='balanced'` algorithmically penalizes the model for misclassifying the minority class. This prevents the model from simply guessing "Noise" every time if the training dataset contains significantly more dismissed articles than kept articles.
* **Convergence Stability:** `max_iter=1000` ensures the gradient descent algorithm converges smoothly even on complex, high-dimensional textual data.

---

## 4. Model Persistence & System Integration

### 4.1 Serialization (`joblib`)
Once `model.fit(X, y)` completes, the trained pipeline (containing both the fitted TF-IDF vocabulary and the Logistic Regression weights) is serialized using `joblib.dump()`. 
* **Target Output:** The binary file is saved explicitly to `src/ml_model.pkl`.

### 4.2 Application Integration
Within the broader ecosystem:
* **Trigger Mechanism (`scheduler.py` & `app.py`):** The `train()` function can be executed on-demand via the UI by an administrator, or autonomously via the weekly background cron job to ensure the model naturally adapts to shifting threat landscapes over time.
* **Inference Consumption (`logic.py`):** The `HybridScorer` class dynamically loads `src/ml_model.pkl` into memory. Every new RSS article fetched by the system is subsequently passed into `model.predict_proba()` to generate the AI Confidence Score used for automated threat triage and prioritization.
