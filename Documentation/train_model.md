# Enterprise Architecture & Algorithmic Specification: `src/train_model.py`

## 1. Executive Overview

The `src/train_model.py` module is the **Machine Learning Pipeline and Feedback Loop Engine** for the Intelligence Fusion Center. It is responsible for training the probabilistic text classification model that powers the Artificial Intelligence "Safety Net" utilized in the `logic.py` scoring engine.

Instead of relying on a static, pre-trained model, this module implements a **Human-in-the-Loop (HITL)** reinforcement architecture. It continuously learns from the explicit actions of NOC analysts. When operators click "Keep" or "Dismiss" on an intelligence feed item in the Streamlit UI, this script ingests those decisions to train a bespoke, mathematically weighted model tuned perfectly to the organization's unique threat profile.

---

## 2. Data Ingestion & Preprocessing

### 2.1 The Human-in-the-Loop (HITL) Dataset
The `train()` function begins by opening a database session and querying the `Article` table for ground-truth training data.
* **Filter Criteria:** It strictly pulls articles where `human_feedback` is either `1` (Dismiss/Bad) or `2` (Confirm/Keep). Unrated articles (`0`) are ignored to prevent introducing noise into the training set.
* **Pandas Integration:** The SQLAlchemy query is passed directly into a Pandas DataFrame (`pd.read_sql()`) for highly optimized, vectorized data manipulation.

### 2.2 Volume Safety Threshold
* **Minimum Viable Data (MVD):** Before proceeding, the engine checks if `len(df) < 10`. Machine Learning models require a baseline variance to identify statistical patterns. If an operator attempts to train the model with fewer than 10 labeled articles, the script safely aborts, preventing the generation of an overfitted, highly inaccurate model.

### 2.3 Feature Engineering & Label Mapping
* **Feature Concatenation:** The predictive feature (`X`) is generated by concatenating the article's `title` and `summary`. This maximizes the text corpus available to the vectorizer, providing deeper semantic context than a headline alone.
* **Binary Labeling:** The raw database integers are mapped to standard binary classification labels (`y`):
  * `1` (Dismiss) $\rightarrow$ `0` (Negative Class)
  * `2` (Confirm) $\rightarrow$ `1` (Positive Class)

---

## 3. The Machine Learning Architecture

The module utilizes `scikit-learn` to construct a highly efficient, two-stage NLP pipeline (`make_pipeline`).

### Stage 1: TF-IDF Vectorization
The text data is passed into `TfidfVectorizer(stop_words='english')`.
* **Mechanism:** Converts the raw text into numerical arrays (vectors). It calculates the Term Frequency (how often a word appears in an article) and multiplies it by the Inverse Document Frequency (penalizing words that appear across *all* articles).
* **Optimization:** The `stop_words='english'` parameter automatically strips out low-value conjunctions and articles (e.g., "the", "and", "but"), isolating high-value threat terminology without requiring manual regex grooming.

### Stage 2: Multinomial Naive Bayes Classifier
The vectorized data is fed into a `MultinomialNB()` classifier.
* **Selection Rationale:** While Deep Learning/Neural Networks are powerful, they require massive datasets and high compute overhead. Multinomial Naive Bayes is an enterprise standard for document classification because it is exceptionally fast, lightweight, and performs remarkably well on small, highly specific text corpuses (making it ideal for a newly deployed NOC application that is still building its training dataset).

---

## 4. Model Persistence & System Integration

### 4.1 Serialization (`joblib`)
Once `model.fit(X, y)` completes, the trained pipeline (containing both the fitted TF-IDF vocabulary and the Naive Bayes probability weights) is serialized using `joblib.dump()`. 
* **Target Output:** The binary file is saved to `src/ml_model.pkl`.

### 4.2 Application Integration
Within the broader ecosystem:
* **Trigger Mechanism (`app.py`):** In the "Settings & Admin" $\rightarrow$ "ML Training" tab, an administrator can click the **"Retrain Model Now"** button. This executes the `train()` function on demand, allowing the system to rapidly adapt to a new zero-day campaign immediately after operators tag a few relevant articles.
* **Inference Consumption (`logic.py`):** The `HybridScorer` class dynamically loads `ml_model.pkl` into memory. Every new RSS article fetched by the system is subsequently passed into `model.predict_proba()` to generate the AI Confidence Score used for threat triage.
