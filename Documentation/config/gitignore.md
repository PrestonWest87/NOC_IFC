# .gitignore — Git Exclusion Rules

**Path:** `/home/weast/docker/NOC_IFC/.gitignore`

## Purpose

Prevents sensitive, generated, and environment-specific files from being tracked by Git. Organized by category for maintainability.

## Entries

### Environment & Secrets

| Pattern | Purpose |
|---------|---------|
| `.env` | Local environment variables containing database URLs, API keys, email credentials, and secrets. **Must never be committed.** |

### Runtime Data

| Pattern | Purpose |
|---------|---------|
| `data/` | Runtime persistent data — SQLite database files, cached API responses, and temporary artifacts. Recreated by the application at runtime. |

### Python Bytecode & Cache

| Pattern | Purpose |
|---------|---------|
| `src/__pycache__/` | Python bytecode cache directory in the source tree. |
| `*.pyc` | Compiled Python bytecode files (any depth). |
| `*.pyo` | Optimized Python bytecode files (legacy Python 2). |
| `__pycache__/` | Python bytecode cache directories at any depth. |

### IDE & Editor

| Pattern | Purpose |
|---------|---------|
| `.vscode/` | Visual Studio Code workspace settings, launch configurations, and extensions. Machine-specific. |
| `.idea/` | JetBrains IDE (IntelliJ, PyCharm, WebStorm) project settings. Machine-specific. |

### OS Files

| Pattern | Purpose |
|---------|---------|
| `.DS_Store` | macOS Finder folder metadata and icon positions. |
| `Thumbs.db` | Windows Explorer thumbnail cache. |

### Node.js / Frontend Build Artifacts

| Pattern | Purpose |
|---------|---------|
| `web/node_modules/` | Installed Node.js dependencies. Recreated by `npm install` / `npm ci`. |
| `web/dist/` | Vite production build output (bundled JS, CSS, assets). Generated by `npm run build`. |
| `*.tsbuildinfo` | TypeScript incremental build info files (`.tsbuildinfo` — generated by `tsc -b`). |

## Usage

This file is read automatically by Git. No action is required beyond keeping it updated as the project evolves.

To verify what is being ignored:

```bash
git check-ignore -v path/to/file
```

To temporarily override the ignore rule for a specific file (e.g., force-add a `.env` template):

```bash
git add -f .env.example
git add -f .env
```
